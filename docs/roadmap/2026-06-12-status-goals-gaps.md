### 2.3 글자 크기의 실제 상태 (2판 정정)
- 0.7대 rem 선언이 30곳 검색되지만, **후행 공통 CSS가 대부분을 0.875rem/1rem `!important`로 덮는다**(style.css 후반부). 따라서 "30곳 상향 개발"은 불필요.
- 실제로 14px 미만으로 적용되는 정보성 요소는 **약 6~10개**: D-day 단위 라벨, 주소록 기수 배지, 프로그램 시간 표기 등. 이 선택자들만 정리하면 끝.
- 장식성 텍스트(푸터 저작권 등)는 작아도 무방하므로 대상에서 제외.

## 3. 남은 일

결론부터: **지금 대규모 기능 개발을 더 하는 것은 맞지 않다.** 코드는 외부 서비스를 기다리는 상태고,
남은 개발의 본질은 "새 기능"이 아니라 **연결 후 드러나는 통합 결함과 실기기 검증 결과에 대응하는 일**이다.

### 3.1 개발 — 소규모 정리만
- [ ] 실제 적용되는 작은 글씨 6~10개 선택자 정리 (2.3 목록)
- [ ] 404.html에 favicon 추가 (선택)
- [ ] Lighthouse 모바일 측정 → **명확한 문제만** 수정 (점수 쫓기 금지)
- [ ] 이후는 통합·검증 대응: 운영 시트/폼/결제 실연결 시 나오는 결함 처리
- admin.html: **개발하지 않는다.** 토큰 관리 위험 대비 효용이 낮다. 행사 후에도 필요성이 실제로 확인될 때만 재검토.

### 3.2 연결 — 코드가 기다리는 빈 값들 (현재 전부 비어 있음)
`SURVEYS`(설문 3종) · `OPS_SHEETS`(운영 시트 6탭) · `PAYMENTS`(결제) · `MAP_LINKS`(지도) · `CONTACT`(전화·오픈채팅) · `EVENT_DETAILS`(라이브·환불 등) · videos.json
- [ ] 운영 시트 개설 → OPS_SHEETS 기입 → **왕복 테스트** (가이드: docs/operations-sheet-guide.md)
- [ ] 구글폼 3종 생성 → SURVEYS 기입 (참석 조사 최우선, 3.4 결정에 종속)
- [ ] YouTube 업로드 (mp4 25개 준비됨) → videos.json
- [ ] 후원사 등록 — 0610 후보 7개사: 컴퍼니케이파트너스·라이온로보틱스·야무지니로보틱스·한국단자공업·노비텍·싸이로드·에스트래픽
- [ ] CONTACT(총무 전화·오픈채팅), 지도 링크, 환불 규정

### 3.3 검증 — 실기기 1회전
