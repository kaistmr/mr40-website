# 40주년 설문 2종 (동잠 주문 · 동문 근황/연락처) 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 이벤터스 전환 후 남은 설문 2종(동잠 주문, 동문 근황·연락처)을 구글폼 임베드 패턴으로 사이트에 배선하고, 운영자가 폼을 만들어 연결할 수 있게 한다.

**Architecture:** 기존 서버리스 구조 유지 — 구글폼을 `survey.html?id=<id>`로 임베드하고, 응답은 구글시트에 쌓인다. 사이트 코드 변경은 `config.js`의 `SURVEYS` 배열·`SITE_URL`과 운영 시트 템플릿 CSV뿐이며, 실제 폼 생성·문항 입력은 운영자가 본 계획의 문항 명세를 복붙해 수행한다.

**Tech Stack:** Vanilla JS, 정적 HTML, 구글폼/구글시트(CSV 게시), GitHub Pages. 빌드 과정 없음 — push 시 자동 배포.

## Global Constraints

- 쓰기 가능 영역은 `40th_website/` 내부뿐. 외부 폴더는 읽기 전용.
- 백엔드/서버 추가 금지 — 서버리스(구글폼·구글시트·CSV) 유지.
- 모든 사용자 노출 문구는 한국어.
- `SURVEYS` 항목 스키마: `id, icon, title, desc, url, active, prefill_generation_key`.
- 운영 시트 통계 열: `group, label, value, sort, public` (헤더 변경 금지).
- 운영 시트 설문링크 열: `id, icon, title, desc, url, active, prefill_generation_key` (헤더 변경 금지).
- 전체 테스트는 항상 그린 유지:
  `cd scripts && python3 -m unittest test_build_gallery test_site_static -v && node test_ops.js`
- 개인정보(전화·이메일)는 공개 범위·동의를 항상 고려.

````markdown
## 공통 설정
- 설정 → 응답 → "이메일 주소 수집" 끄기(이름·기수로 식별).

## 폼 ① 동잠(단체복) 주문

1. (필수) **단체복 구매 여부** — 객관식: `구매합니다.` / `구매하지 않습니다.`

3. **이름(본명)** — 단답형.
4. **성함(한자)** — 장문형. 안내: "오른쪽 손목 부근에 새겨질 예정입니다. 한자 표기를 적어주세요. 구매하시는 분은 꼭 답변 부탁드립니다."
5. **성함(영문)** — 단답형. 안내: "왼쪽 손목 부근에 새겨질 예정입니다. 예: Y.S. An. 구매하시는 분은 꼭 답변 부탁드립니다."
6. (필수) **사이즈** — 객관식: `XS` / `S` / `M` / `L` / `XL` / `2XL` / `3XL`.
7. (필수) **수량** — 단답형(숫자).
8. (필수) **수령 방법** — 객관식: `행사장 수령` / `택배(주소 별도 안내)`.
9. **연락처** — 단답형(휴대폰).
10. **비고** — 장문형(선택).

마지막 섹션 설명에 결제 안내: "입금은 주문 수합 후 총액이 확정되면 별도로 안내드립니다."
````

```js
const assert = require("node:assert/strict");
const fs = require("node:fs");
const path = require("node:path");

const src = fs.readFileSync(path.join(__dirname, "..", "config.js"), "utf8");
const CONFIG = new Function(src + "; return CONFIG;")();

const ids = CONFIG.SURVEYS.map((s) => s.id);
assert.ok(!ids.includes("voice"), "voice 설문은 방명록으로 흡수되어 제거되어야 함");

const attend = CONFIG.SURVEYS.find((s) => s.id === "attend");
assert.equal(attend.active, false, "참석 조사는 이벤터스 전환으로 active:false 여야 함");

for (const id of ["jacket", "contact"]) {
  const s = CONFIG.SURVEYS.find((x) => x.id === id);
  for (const key of ["id", "icon", "title", "desc", "url", "active", "prefill_generation_key"]) {
    assert.ok(key in s, `${id} 항목에 ${key} 필드가 있어야 함`);

assert.ok(!/mosw626/i.test(CONFIG.SITE_URL), "SITE_URL에 옛 개인 레포(mosw626)가 남아있으면 안 됨");
assert.ok(/kaistmr\.github\.io/.test(CONFIG.SITE_URL), "SITE_URL은 kaistmr 주소여야 함");

console.log("config surveys OK:", ids.join(","));
```

## 메모: 운영자 수동 단계(코드 외)

- 연락처 응답 → 운영자 검토 → `MEMBERS_CSV` 시트 수동 병합(공개동의·공개범위 반영).
