# 공개용 주소록 파생 시트 — 설정 가이드 (운영자)

연락처 설문(동문 근황·연락처) **응답 시트를 주소록에 직접 연결하면 안 된다.**
이유: (1) 열 순서가 주소록과 다르고, (2) "비공개"·미동의자까지 게시 CSV로 전부 노출된다.

그래서 **동의 + 동문 공개를 선택한 행만, 필요한 칸만** 담은 "공개용 주소록" 시트를 따로 만들어
그 시트만 CSV로 게시해 `MEMBERS_CSV_URL`에 연결한다. 원본 응답 시트는 **비공개로 둔다.**

## 주소록(members.html)이 읽는 열 순서

`slice(1)`로 1행(헤더)은 버리므로 **1행은 헤더, 2행부터 데이터**여야 한다.

| 열 | A | B | C | D | E | F | G |
|---|---|---|---|---|---|---|---|
| 의미 | 이름 | 기수 | 소속 | 이메일 | 휴대폰 | 관심사 | 한마디 |

## 응답 시트 열 (동문 근황·연락처 폼 기준)

A 타임스탬프 · B 기수 · C 이름 · D 휴대폰 · E 이메일 · F 주소·근황 ·
G 공개 범위 · H 동의 · I 최종학력 · J 전공 · K 현재 소속·직위 · L 경력요약

## 만드는 법

1. 연락처 응답 스프레드시트에서 **새 시트 탭** 추가 → 이름 `공개주소록`.
2. **A1:G1에 헤더** 입력: `이름  기수  소속  이메일  휴대폰  관심사  한마디`
3. **A2** 셀에 아래 수식(응답 탭 이름을 실제 이름으로 교체 — 보통 `설문지 응답 시트1`):

   ```
   =QUERY('설문지 응답 시트1'!A:L, "select C, B, K, E, D where H = '동의합니다' and G = '동문에게만 공개' and C is not null", 0)
   ```

   → 이름·기수·소속·이메일·휴대폰(A:E)이 채워진다.

4. **G2** 셀에 아래 수식(같은 필터로 한마디만):

   ```
   =QUERY('설문지 응답 시트1'!A:L, "select F where H = '동의합니다' and G = '동문에게만 공개' and C is not null", 0)
   ```

   → 한마디(근황)가 G열에 채워진다. **F열(관심사)은 비워 둔다.**
   두 수식이 같은 필터라 행 순서·개수가 일치하므로 A:E와 G가 정렬된다.

5. `공개주소록` 시트를 **"파일 → 공유 → 웹에 게시 → 공개주소록 시트 → CSV"** 로 게시.
   → 그 CSV 주소를 안윤수에게 전달 → `MEMBERS_CSV_URL`에 연결.

6. **원본 응답 시트(이미 공개 게시한 연락처·동잠 CSV)는 게시 중지(비공개).**

## 동작 확인

- "동문에게만 공개" + "동의합니다" 행만 나오는지, "비공개"/미동의 행이 빠졌는지 시트에서 눈으로 확인.
- 이메일·휴대폰은 주소록 페이지(접속 코드 보호)에 노출되므로, 공개에 동의한 동문만 포함됨을 보장.

## 주의

- 게시 CSV는 URL만 알면 누구나 받을 수 있다(접속 코드는 화면 UI만 가림). 그래서 **반드시 동의·공개 선택자만** 이 시트에 들어가야 한다.
- 동잠 주문 응답은 주소록과 무관(명부가 아닌 주문 정보)하므로 여기 포함하지 않는다.
