# 40주년 설문 2종 (동잠 주문 · 동문 근황/연락처) — 설계

날짜: 2026-06-25
상태: 승인됨 (사용자 확인)

## 목적

이벤터스로 참가 신청·결제가 일원화된 뒤 남은 설문 2종을 정리·구축한다.

- **참석 조사**는 이벤터스와 신청 경로가 중복되어 내림(`active: false`, 완료됨).
- **총회 한마디**는 별도 설문을 만들지 않고 기존 **방명록(guestbook)** 으로 흡수.
- 새로 만드는 설문은 둘: **① 동잠(단체복) 주문**, **② 동문 근황·연락처 업데이트**.

## 배경·결정 사항

- 도구는 기존 인프라와 일치하는 **구글폼 임베드** 방식 유지 (서버리스, 백엔드 없음).
  - `survey.html?id=<id>` 가 `CONFIG.SURVEYS`의 해당 항목 `url`을 임베드하고, `prefill_generation_key`가 있으면 기수(generation)를 자동 prefill.
- 사이트는 **원응답을 자동 집계하지 않는다.** `stats.html`은 미리 집계된 수치 행
  (`group / label / value / sort / public`)을 막대그래프로 표시할 뿐이다.
  → 경력 통계는 **운영자가 집계 → stats 시트** 단계를 거친다.
- 주소록(`members.html`)은 별도 `MEMBERS_CSV_URL` 시트를 읽는다. 연락처 응답은 자동 연결하지 않고
  **운영자가 검토 후 수동 병합**한다(개인정보·중복·오기·공개동의 확인 필요).

## 설문 ① 동잠(단체복) 주문 — `id: "jacket"`

구글폼. 단순 사이즈 주문이 아니라 **손목 각인(이름)** 이 핵심.

문항(* = 필수):

1. **단체복 구매 여부\*** — 구매합니다 / 구매하지 않습니다
   - "구매하지 않습니다" 선택 시 이후 각인·사이즈 문항은 구글폼 섹션 분기로 건너뜀
2. 기수 — `survey.html`이 URL `generation`으로 자동 prefill (키는 `prefill_generation_key`)
3. 이름(본명)
4. **성함(한자)** — 오른쪽 손목 부근 각인용. 한자 표기 안내.
5. **성함(영문)** — 왼쪽 손목 부근 각인용. 예: `Y.S. An`
6. 사이즈 — XS/S/M/L/XL/2XL/3XL (실제 사이즈표는 폼 작성 시 확정)
7. 수량
8. 수령 방법(행사장 수령 / 택배) + 연락처
9. 비고(선택)

결제: **"입금은 주문 수합 후 총액 확정 시 별도 안내"** 를 구글폼 안내문에 명시.
(`survey.html`의 결제 단계 UI는 `id=attend` 전용이므로 jacket에는 뜨지 않음 — 결제 안내는 폼 자체에서 처리하고,
필요하면 jacket 설명에 한 줄 안내를 추가한다. 향후 필요 시 이벤터스 상품으로 전환 여지를 남김.)

## 설문 ② 동문 근황·연락처 업데이트 — `id: "contact"`

구글폼 하나. 한 폼에서 받되 **소비처가 둘로 갈린다.**

### A. 연락처 → 주소록 (수동 큐레이션)
- 휴대폰 / 이메일 / (선택)주소·근황 한마디
- **공개범위 선택**(동문에게만 공개 / 비공개)
- **개인정보 수집·이용 동의\***(필수 체크)
- 반영: 응답 시트 → 운영자 검토 → `MEMBERS_CSV` 시트 병합.

### B. 졸업 후 진로·경력 → 40년 통계(`stats.html`)
- **최종 학력\*** — 학사 / 석사 / 박사 / 포닥 / 기타
- **전공 분야\*** — 석·박사는 세부 전공
- **현재 소속 및 직위\*** — 예: OO전자 책임연구원, OO대학교 교수, 벤처 창업
- **주요 경력 요약\*** — 40주년 기념 자료용 자유 서술
- 파이프라인: 원응답 → 운영자 집계 → `stats` 시트(`group/label/value/sort/public`) → `stats.html` 막대그래프.

### 개인정보(PIPA)
- 폼 상단에 수집 목적·항목·보관 기간·공개 범위 고지, 동의 필수.
- 통계는 **공개 가능한 집계 수치만** 노출(개인 식별 정보 비노출).

## config.js 반영

- `SURVEYS` 배열:
  - `attend`: `active: false` 유지
  - `voice`: 제거(방명록으로 대체)
  - `jacket`: 신규 추가 (icon/title/desc/url/active/prefill_generation_key)
  - `contact`: 기존 id 재활용, `url`·`prefill_generation_key` 채움
- `OPS_SHEETS.stats`: 집계 시트 CSV URL 입력 시 통계 자동 노출.

## 역할 분담

- **구글폼 생성·문항 입력**: 사용자(구글 계정 권한). 본 문서의 문항 명세를 복붙해 구성.
- **사이트 반영**: 받은 폼 URL을 `config.js`에 연결, 결제 안내 문구 정리, stats 시트 양식(열 머리글) 제공.

## 변경 파일

- `config.js` — `SURVEYS`/`OPS_SHEETS` 갱신 (URL은 폼 생성 후 입력)
- `survey.html` — 필요 시 jacket 안내 문구(결제 추후 안내) 보강. 기존 임베드·prefill 로직 재사용.
- (문서) 구글폼 문항 명세·stats 시트 양식 안내

## 범위 외 (YAGNI)

- 동잠 결제의 사이트 내 처리(추후 별도 안내 또는 이벤터스 전환)
- 연락처 응답의 주소록 자동 반영(수동 큐레이션 유지)
- 원응답 자동 집계·실시간 통계(운영자 집계 단계 유지)
- 총회 한마디 전용 설문(방명록으로 대체)
